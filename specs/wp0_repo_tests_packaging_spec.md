# WP-0 spec — repository, tests, packaging, CI (v1)

**Goal.** Turn `columna_core` v0.7.7 from a folder of modules + demo scripts into an installable,
CI-tested, Apache-2.0 open-source package inside the Datumwise monorepo — with **zero behavior
changes**. This WP is release engineering only.

**Context files (read before coding).**
- `columna_core/` tree (the library, 12 modules, ~2,430 lines) and its `README.md` (the version
  history is the de-facto changelog — preserve it).
- `adr_031_columna_core.md`, `adr_032_columna_engine.md` (locked vocabulary: planner vs column
  engine; the four outcomes; two-level contract. Do not rename anything they name.)
- The 11 demo suites at repo top level (`coanchor_demo.py`, `confine_demo.py`, `holistic_demo.py`,
  `locus_demo.py`, `operator_umbrella_demo.py`, `projection_demo.py`, `types_demo.py`,
  `universe_check_demo.py`, `hll_case_study_demo.py`, plus `build_benchmark.py` /
  `parse_benchmark.py`) — together 124 assert-style checks.

## Packaging (`packages/columna-core/pyproject.toml`)

- Name `columna-core`, version `0.7.7`, `requires-python >= 3.10`, Apache-2.0.
- Dependencies, pinned as ranges: `polars`, `duckdb`, `datasketches`. (All three are hard imports
  today — `engine.py` imports `sketch.py` at module top, `__init__` imports `connector`. Making
  `duckdb`/`datasketches` optional extras is **WP-1.1 scope**, where the import graph is
  restructured for the PolarsConnector; do NOT attempt it here.)
- Entry points: none in this WP.
- Build backend: hatchling (or implementer's standard); wheel + sdist must build.

## Test conversion (the bulk of the work)

Principles:
1. **Demos stay.** Each demo script remains in `demos/`, runnable, unchanged in what it prints —
   they are executable documentation and the ADR-cited proof scripts.
2. **Tests mirror demos.** For each demo suite create `tests/test_<name>.py` that exercises the
   same 124 checks as proper pytest tests (one test function per logical check-group; asserts with
   messages preserved). No new coverage is required; no check may be dropped — the acceptance
   number is **124 pytest-collected assertions minimum**, mapped in a table in the PR description
   (demo check → test id).
3. **Data fixtures.** Demos that run against the real benchmark warehouse must gain a
   *small bundled fixture* path: a `tests/fixtures/` mini-warehouse (generated by a
   `tests/fixtures/make_fixture.py` script, checked-in parquet ≤ 2 MB total, same schema as the
   benchmark warehouse). Tests run against the fixture by default. The full-warehouse validation
   (`build_benchmark.py`, 10/10 real-data proofs incl. the 299,934-txn checks) becomes an opt-in
   marked test: `@pytest.mark.warehouse`, skipped unless `COLUMNA_BENCH_WAREHOUSE` env var points
   at the instance directory.
4. **conftest.py** provides: fixture-warehouse `DuckDBConnector`, a parsed demo Manifold
   (`benchmark.cml` — move it to `tests/fixtures/` and `demos/` keeps a copy), and a
   `ManifoldServer` fixture.
5. Numeric assertions against the fixture must be computed by the fixture generator and written
   into a `fixtures/expected.json` — never hand-typed constants.

## CI (`.github/workflows/ci.yml`)

- Matrix: Python 3.10 / 3.11 / 3.12, ubuntu-latest.
- Steps: install, `ruff check` (config permissive: line-length 110, no style crusade over the
  existing code — E/F errors only), `pytest -q`, build wheel.
- The `warehouse`-marked tests are excluded in CI.

## Invariants (violating any = rejected PR)

- No module renamed; no public symbol renamed; no behavior change (`git diff` on the 12 modules
  should be import-path moves only).
- Demo scripts still run from `demos/` against the fixture warehouse path or real one.
- ADR vocabulary intact in all new prose (planner, column engine, the four outcomes, Manifold).

## Acceptance

1. `pip install -e packages/columna-core` then `python -c "import columna_core"` succeeds.
2. `pytest packages/columna-core -q` green, ≥124 assertions collected, no network, < 90 s.
3. `pytest -m warehouse` with env var set reproduces the 10/10 real-data proofs.
4. CI green on the matrix; wheel builds.
5. Check-mapping table present in PR.
