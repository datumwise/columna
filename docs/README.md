# docs/ — the Columna manuals

The authoritative home of the Columna manuals. **The merge that published these files is the
publication** (ADR-034 D5 + ADR-035 D4; residency ruling 2026-07-14); nothing manual-shaped travels by
upload after that.

## The manuals
- [`columna_reference_manual_5e.md`](columna_reference_manual_5e.md) — the reference manual (5th ed.):
  the object model, the definition language (Chapter 26), the Certificate layer (Part VI).
- [`columna_framework_manual_6e.md`](columna_framework_manual_6e.md) — the framework manual (6th ed.).
- [`frame_ql_manual_v1.md`](frame_ql_manual_v1.md) — the Frame-QL query-language manual (v1).

Each construct in the reference manual's Chapter 26 carries a shipping status mark
(**[SHIPPED vX.Y]** · **[SCHEDULED — <WP>]** · **[ROADMAP — <note>]**); unmarked constructs do not
exist (ADR-034; the honest-hatching idiom). The Frame-QL manual documents the **shipped envelope
grammar** as canonical (`columns @ anchor`); its historical Coframe-QL canonical form is retained as a
lineage appendix (ADR-035).

## tools/ — the regeneration harness (the manual's integrity rule)
Every executable Frame-QL example is a ```frameql query block followed by a ```frameql-output block
that is **machine-generated by running the query against the shipped package** — the site's build-time
integrity rule, extended to the manual. See [`tools/regen_examples.py`](tools/regen_examples.py):

    python docs/tools/regen_examples.py            # rewrite every frameql-output block
    python docs/tools/regen_examples.py --check     # CI: fail on any run error or output drift

"The manual is structurally incapable of documenting a query it did not run." CI runs `--check`
(`.github/workflows/docs.yml`); an example that stops parsing, errors, or whose answer drifts fails the
build. `tools/SAMPLE_regeneration.md` is the harness's own smoke fixture.
