## 0. Headline

The shipped touch seam is a **scheme-branch chokepoint**: only two sites in the whole engine
ever branch on the face scheme — `parser._p_faces` (the deferral gate) and
`engine._resolve_touch` (the basis + monoid/VALUE gates). Everything else — `Face`,
`find_faced`, `parse_faced`, `FaceShape`, the wire projection, the adjudication loop — is
already scheme-agnostic and carries `face.scheme` / `face.selection` through untouched.
`FACE_SCHEMES` **already** = `(touch, assign, alloc)`. So G1 is mostly: relax one parser
line, add two engine resolvers, add two adjudication verdicts. The plumbing already flows.

Pricing lands **in the touch band (~285 LOC) for G1+G4** — *if* G5 ships only its refuse-half.
G5's serve-half ("distinct rides `.touch`") is the wildcard: the recorded run shows it is
today the `unsupported` error the touch gate deferred as **"sketch crossings are post-launch."**
Landing the serve-half means shipping that deferred sketch-crossing (+40–60 LOC and a new reduce
path). **That is the one call for the desk.**

## 1. Design conformance walk (addendum §§1–9)

**§1 — the third universe.** ✅ conformant with one verify-item. `UNIVERSE category_profile =
category BASIS spine` is a degenerate spine (no time axis); the inventory spine proves the
grammar. `MEASURE priority … VALUE priority` / `alloc_weight` are VALUE-style states read at
the base grain. **VERIFY (small):** the spine machinery today always pairs with a `FAMILY`
block (stock has `sum BLOCKED / last ORDER day`); a no-time-axis spine serving a plain VALUE
measure at its own base grain with *no* FAMILY must be exercised — expected trivial (base-grain
read), but it is an unproven path. Driver-lemma holds: category_profile is a spine at the
frontier level, so priority/alloc_weight are lawful drivers.

**§2 — grammar.** ✅ conformant. `_p_faces` already captures `(name, scheme, selection)` via
`(\w+)\s*=\s*(\w+)(?:\s+(.*))?$`; today `selection` ("BY priority") is captured-but-unused and
`scheme != TOUCH` raises the deferral ParseError (`parser.py:240-242`). Plan: relax that gate;
parse `BY <measure-ref>` out of `selection` into the driver slot; resolve the measure-ref
against the manifold at publish (not a physical column). Face names stay operationally inert
(scheme keyword carries the op). **NOTE (naming):** the shipped field is `Face.selection`; the
wire (§5) calls it `driver`. No fight — store the measure-ref in `selection`, project it as
`driver` (null when `''`). Flagged so the desk isn't surprised by the field name in the diff.

**§3 — adjudication.** ✅ conformant, real work. `_prove_face` currently mints timeless
`VERIFIED` for touch and does **not** branch on scheme (`adjudication.py:658-671`). Assign/alloc
proofs **touch data** (they must serve the driver to check distinctness / non-negativity), so
their verdicts are **CORROBORATED**, not VERIFIED — consistent with the shipped License kernel.
Acyclicity (the face→driver→universe→face DAG) is **new machinery**; for 0.12's demo it is
trivially acyclic (drivers live on category_profile, a spine with no faces), but the graph
builder + cycle-detector must exist and fail-closed naming the cycle. Licensing unchanged
(crossed measures events-only; drivers lawful on spine) — matches the shipped events-only mint.

**§4 — engine.** ✅ conformant with **one design gap to ratify.** `_resolve_assign` restricts
the bridge to each product's top-ranked pair (single-row join, no multiply); `_resolve_alloc`
join-multiplies by the per-product-normalized driver. Both reuse the touch seam's
serve-driver-at-grain + coverage + lawful-zero scaffolding; `_resolve_touch` untouched.
**⚠ DESIGN GAP (needs a ruling): ASSIGN's rank direction.** "take-top" is undefined as max-of-
driver vs min-of-driver. The demo authors `priority` as distinct ints 1..12 and the DRAFT copy
says "top-priority category" — which reads as *priority 1 = top* (take-**min**). The engine
needs the convention pinned. Recommend: **ASSIGN = take-max of the driver**, and author
`priority` so higher = more important (or explicitly define ASSIGN as take-min-rank and keep
1=top). Either is fine; it must be *chosen* so the copy and the code agree. Flagged for the desk.

**§5 — wire (additive).** ✅ conformant, `contract_version` stays `"1"`. `faces[]` gains
`scheme` (already projected) + `driver` (new, from `selection`). Assign adds
`memberships_unrepresented` (the shadow; **grounded = 270**, see below). Alloc adds
`reconciliation: {crossed_total, base_total, delta, tolerance, status}`. Both are additive
payloads + new `disclosure_wire` rows; touch's `multi_counted` unchanged.

**§6 — G4 chain guard.** ✅ conformant, cheap. Today `_resolve_touch` has a generic single-faced
guard (`engine.py:258-262`) that already refuses multi-faced asks as `unsupported`. G4 = replace
that generic refusal with the **named** guard message when an ask crosses two declared faces in
sequence. **VERIFY (small):** pin the chain-detection predicate (what counts as "already
crossed" — a second faced coordinate in the same anchor vs a faced coord over an already-crossed
sub-result). For 0.12 the >1-faced-token case covers the demo surface.

**§7 — G5 distinct-at-faces.** ⚠ **conformant in intent, but the serve-half is the off-ramp
finding (see §6 of this doc).** Verify done (recorded run below). The refuse-half
(`.assign`/`.alloc` over a distinct-class measure → REFUSE) is cheap and clean. The serve-half
("distinct **may ride** `.touch` with the dedupe/overlap disclosure") requires building the
per-member sketch reduce that the shipped touch gate explicitly defers: *"touch crosses additive
(monoid VALUE) measures only in v1 … sketch crossings are post-launch."* That is a genuine scope
decision, not a rule assertion. **Desk ruling requested.**

**§8 — exhibits / menu / manual / ledger.** ✅ conformant. E11/E12 seeds ride the standing
recapture pipeline; the bare-ask menu (`planner.py:204-205`) already lists every declared face,
so it goes three-deep automatically once assign/alloc parse. Manual triad chapter replaces the
deferred-refusal section. Ledger gets the dated re-sequencing note (DRAFT).

**§9 — release + discipline.** ✅ 0.12.0 across core/server/meta, publish-first flip, byte-stable
except where E11/E12 add. This proposal satisfies the proposal-first gate.

## 4. G5 verification transcript (recorded run, not reasoning)

Run against the **shipped** Cascadia demo (`columna_server.demo.demo_store()` →
`tools.query(store, "cascadia", …)`), 2026-07-24, editable install of core+server at current
main. **Verbatim outputs:**

```
QUERY: SELECT buyers AT {category.touch}
outcome: error
  col buyers: status=error
    no_result: kind=error, reason="unsupported",
      detail="touch crosses additive (monoid VALUE) measures only in v1 —
              'buyers.distinct' is not (ordered/holistic/sketch crossings are post-launch)"
```

Context runs (to characterize the landscape):

```
QUERY: SELECT revenue AT {category.touch}          # E10 sanity — additive touch
outcome: disclose · revenue served · multi_counted (over_count) + coverage 600/600

QUERY: SELECT buyers AT {category}                 # bare distinct at the M:N frontier
outcome: clarify · reason=non_functional_transport (the face menu)

QUERY: SELECT buyers AT {region}                   # distinct on an aligned M:1 grain
outcome: disclose · buyers served · "buyers.distinct: HLL distinct estimate" (SKETCH)
```

**What this establishes:** `buyers` is a **SKETCH** measure (HLL, `witness=SKETCH`, `is_monoid=
True`). The touch gate (`engine.py:275-279`) admits **VALUE monoids only** and refuses SKETCH as
`unsupported`, by design ("post-launch"). So today `buyers AT {category.touch}` is **neither a
served-with-overlap frame nor a clean domain refuse — it is a blanket `unsupported` error.** The
addendum's G5 serve-rule ("distinct **may ride** `.touch` with the dedupe/overlap disclosure")
therefore requires *implementing the sketch crossing*: relax the gate for SKETCH, compute
**per-category distinct** (merge the HLL sketch per member, estimate) instead of monoid-sum, and
emit the cross-member-overlap disclosure (a customer buying in ≥2 categories counts in each; the
summed per-category distinct exceeds the grand distinct). This is buildable and sensible — the
HLL machinery already exists — but it is the deferred post-launch item, not a one-liner.

**The three adjudication-failure cases (all fail-closed, publish-time):**
1. **Tied top rank (assign).** Fixture: a product with two categories of equal `priority` →
   `_prove_face(assign)` fails, message names the tied categories **and** the affected product(s).
2. **Zero-sum driver (alloc).** Fixture: a product whose every category carries `alloc_weight=0`
   → per-member sum = 0 → undefined split → the face **does not publish** (fail-closed, named).
3. **Dependency cycle (acyclicity).** Fixture: a face whose driver measure's universe would
   require a face that (transitively) requires the first → DAG check fails, **names the cycle.**
   (Contrived — 0.12's demo is acyclic — but the machinery must reject it.)

**G5 tests:** refuse-half — `buyers AT {category.primary}` / `{category.split}` REFUSE with the
distinct-class message. Serve-half (if ruled in) — `buyers AT {category.touch}` serves

## 6. Structural surprise → the standing off-ramp

**One finding returns to the desk before build:**

> **G5's serve-half ships the sketch crossing the touch gate deferred as "post-launch."**
> The recorded run proves `buyers AT {category.touch}` is today an `unsupported` error, not a
> servable frame. Making distinct "ride `.touch` with disclosure" = building the per-member
> sketch reduce + relaxing the VALUE-only gate (+~50 LOC of *new* engine mechanics). Two clean
> options:
>
> - **(A) Full G5 now** — implement distinct-rides-`.touch` serve + the refuse-half. Total
>   ~303 LOC; ~50 of it is the new sketch-crossing path (higher uncertainty).
> - **(B) G5 refuse-half now, serve-half deferred** — `.assign`/`.alloc` over distinct REFUSE
>   (ships the honest guard); `buyers AT {category.touch}` keeps today's `unsupported` (or gets a
>   cleaner "sketch crossing not yet licensed" message). Total ~253 LOC, all low-risk. The
>   serve-half rides 0.13 with the sketch-crossing work.
>
> The addendum reads as intending (A) ("may ride `.touch`"); the shipped gate's own words defer
> it. **Desk's call.**

**Two smaller ratifications before build (not off-ramps, just gaps):**
- **ASSIGN rank direction** (§1/§4 above): take-max vs take-min of the driver — pin it so the
  demo's `priority` authoring and the "top-priority" copy agree.
- **Degenerate-spine VALUE path** (§1): a no-FAMILY, no-time spine measure is an unexercised
  path; expected trivial, will prove it first thing in build.
